1.软件的制作时间应该像蛋糕 如果耗费时间远远大于评估时间 十有八九是会有问题的
2.蛋糕的体积很小 软件的规模也不应该很大 小体积可以很容易的进行试错 小体积意味着可以快速的大量迭代
3.蛋糕有消费者 软件也应该有消费者

总之 把软件像成一块蛋糕 可以想通很多事情
