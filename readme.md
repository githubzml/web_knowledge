Cookie 验证流程 存量较小 存储登录状态 用户资料 小文本类 浏览器通常会自行添加 会造成csrf攻击漏洞 不超过4k
01. 用户输入登录凭证
02. 服务器验证凭据是否正确，并创建对话，然后会把数据存储在数据库中
03. 具有会话id的cookie被放置在浏览器中
04. 后续请求 服务器会验证会话id
05. 一旦用户登出 服务器和客户端会同时销毁该会话

Cookie + Session 客服增多 服务器器压力大 需要双向存储

03. 客户端负责存储token

05. 服务器端对token有效进行处理
06. 一旦登出 客户端销毁token
